# Git 子模块使用说明

## 概述
本文档说明如何将 `daynote` 目录配置为 Git 子模块，使其能够独立管理同时也作为主项目的一部分。

## 配置步骤

### 1. 初始化主仓库
```bash
cd c:\Users\P30015874206\Desktop\note
git init
```

### 2. 创建独立的 daynote 仓库
```bash
cd c:\Users\P30015874206\Desktop\note\daynote
git init
```

### 3. 添加并提交 daynote 内容
```bash
cd c:\Users\P30015874206\Desktop\note\daynote
git add .
git commit -m "Initial commit for daynote submodule"
```

### 4. 将 daynote 添加为主仓库的子模块
```bash
cd c:\Users\P30015874206\Desktop\note
git submodule add ./daynote
git commit -m "Add daynote as submodule"
```

## 子模块日常操作

### 克隆包含子模块的仓库
```bash
git clone --recurse-submodules <repository-url>
```

### 更新子模块
```bash
git submodule update --remote
```

### 在子模块中提交更改
```bash
cd daynote
# 进行修改后
git add .
git commit -m "Commit message"
cd ..
git add daynote
git commit -m "Update daynote submodule"
```

## 注意事项
1. 子模块有自己的 Git 历史记录
2. 更新子模块后需要在主仓库中提交引用更新
3. 其他开发者克隆仓库时需要使用 `--recurse-submodules` 参数
4. 使用 `.gitignore` 文件忽略不需要版本控制的文件和目录，如历史记录、虚拟环境等
