# 个人笔记系统使用说明书

## 系统概述

本系统是一个基于 VSCode 的个人笔记管理系统，旨在帮助您高效地记录日常工作、自动生成日报以及评估工作状态。

## 目录结构

```
note/
├── daynote/                 # 日常笔记目录
│   ├── 20251222/           # 以日期命名的子目录
│   └── model/              # 模板目录
│       ├── 灵感.md          # 灵感笔记模板
│       └── 日报.md          # 日报模板
├── 日报/                   # 自动生成的日报目录
├── scripts/                # 脚本目录
│   └── create_note.py      # 创建笔记的Python脚本
└── .vscode/                # VSCode配置目录
    ├── tasks.json          # 任务配置
    ├── keybindings.json    # 快捷键配置
    └── settings.json       # 设置配置
```

## 功能特性

### 1. 自动创建日期目录
系统会在需要时自动创建以 `YYYYMMDD` 格式命名的子目录，例如 `20251222`。

### 2. 模板化笔记创建
通过预设模板快速创建标准化的笔记文件：
- 灵感笔记：记录日常想法、创意、问题和解决方案
- 日报：总结当日工作内容和计划

### 3. 快捷键操作
提供快捷键快速创建各类笔记，无需手动操作。

### 4. 自动替换日期占位符
创建笔记时自动将模板中的 `{{date}}` 替换为当前日期。

## 使用方法

### 快捷键操作（推荐）
- `Ctrl+Alt+I`：创建灵感笔记
- `Ctrl+Alt+R`：创建日报
- `Ctrl+Alt+C`：创建自定义类型笔记

**注意：如果快捷键操作失败，请使用以下替代方法**

### 命令面板操作
1. 按 `Ctrl+Shift+P` 打开 VSCode 命令面板
2. 输入 "Tasks: Run Task"
3. 选择以下任务之一：
   - `create-inspiration-note`：创建灵感笔记
   - `create-report-note`：创建日报
   - `create-custom-note`：创建自定义类型笔记

### 命令行操作（替代方法）
如果快捷键和命令面板操作失败，可以使用命令行方式：

1. 打开终端/命令提示符
2. 运行创建笔记脚本（无需导航到项目根目录）：
   - 创建灵感笔记：`python scripts/create_note.py --template 灵感`
   - 创建日报：`python scripts/create_note.py --template 日报`
   - 创建自定义类型笔记：`python scripts/create_note.py --template [类型名]`

### 手动创建笔记（备用方法）
如果以上方法都不可用，可以手动创建笔记：

1. 在 `daynote/YYYYMMDD/` 目录下创建新的 `.md` 文件
2. 从 `daynote/model/` 目录复制相应模板内容
3. 手动替换模板中的 `{{date}}` 占位符为当前日期

## 文件生成位置

- 灵感笔记：`daynote/YYYYMMDD/灵感-HHMMSS.md`
- 日报：`日报/日报-YYYY-MM-DD.md`
- 自定义笔记：`daynote/YYYYMMDD/{类型}-HHMMSS.md`

## 模板说明

### 灵感笔记模板
包含以下章节：
- 创意点
- 遇到的问题
- 解决方案
- 学到的知识

### 日报模板
包含以下章节：
- 主线任务检查机制
- 今日要点回顾
- 核心总结

## 状态评估机制

系统会根据当日笔记的字数来粗略评估工作状态：
- 字数 > 500：充实
- 字数 > 200：一般
- 字数 > 50：较轻松
- 其他：轻松

## 自动化日报生成

系统可通过 Windows 任务计划程序设置定时任务，在每天 17:00 自动生成日报，汇总当日所有笔记内容。

## Git 子模块管理

`daynote` 目录已配置为 Git 子模块，可以独立管理其版本历史：

1. 子模块拥有独立的 Git 历史记录
2. 可以在 `daynote` 目录中直接进行 Git 操作
3. 更新子模块后需要在主仓库中提交引用更新

### 子模块日常操作

- 克隆包含子模块的仓库：`git clone --recurse-submodules <repository-url>`
- 更新子模块：`git submodule update --remote`
- 在子模块中提交更改后，需在主仓库中也提交一次以更新引用

## 注意事项

1. 确保已安装 Python 3.x 环境
2. 确保 Python 已添加到系统 PATH 环境变量中
3. 如需修改模板内容，请编辑 `daynote/model/` 目录下的相应模板文件
4. 如需添加新的模板类型，请修改 `scripts/create_note.py` 脚本
5. 其他开发者克隆仓库时需要使用 `--recurse-submodules` 参数以正确初始化子模块
